# Security Policy

`peekabo` is intended for passive analysis of wireless traffic metadata in environments where the operator is authorized to capture and analyze that traffic.

The project must not add features that decrypt traffic, inject frames, transmit probes, exploit networks, or inspect higher-layer payloads.

## Reporting Sensitive Issues

Do not post private packet captures, MAC addresses, credentials, or sensitive network details in public issues. If the repository has private vulnerability reporting enabled, use GitHub's private reporting flow. Otherwise, open a public issue with a minimal sanitized description and offer to share sensitive details privately with a maintainer.

## Supported Versions

The current `main` branch is the only supported development line until versioned releases are established.
